Alpine.js Code:

<div x-data="{ count: 0 }">
    <p x-text="count"></p>
    <button @click="count++">+</button>
</div>

Pros: No build step, declarative, minimal syntax

Vue.js - Reactive & Component-Based

Vue.js Code (Composition API):

const { createApp, ref } = Vue
createApp({
  setup() {
    const count = ref(0)
    return { count }
  }
}).mount('#app')

Pros: Reactive data, component system, excellent DX

React - State-Driven

React Code (Hooks):

function Counter() {
  const [count, setCount] = React.useState(0)
  return (
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  )
}

Pros: Explicit state management, huge ecosystem, JSX flexibility

🔥 Code Comparison - Same Simple Counter

Look at how much code each framework requires for the EXACT same functionality:

Alpine.js

3 Lines 
<div x-data="{ count: 0 }">
    <p x-text="count"></p>
    <button @click="count++">+</button>
</div>

✅ Everything in HTML attributes - super compact!

Vue.js

10 Lines 
<template>
    <div>
        <p>{{ count }}</p>
        <button @click="count++">+</button>
    </div>
</template>

<script setup>
import { ref } from 'vue'
const count = ref(0)
</script>

📦 Separate template + script sections

React

8 Lines 
function Counter() {
    const [count, setCount] = useState(0)
    
    return (
        <div>
            <p>{count}</p>
            <button onClick={() => setCount(count + 1)}>+</button>
        </div>
    )
}

⚙️ Explicit state management with setters

jQuery

12 Lines 
<!-- HTML -->
<div>
    <p id="count">0</p>
    <button id="plus">+</button>
</div>

<script>
let count = 0
$('#plus').click(() => {
    count++
    $('#count').text(count)
})
</script>

🔧 Manual DOM updates + separate HTML/JS

📊 The Verdict:

Alpine.js: 70% less code than others - everything lives in HTML attributes

Vue.js: Clean separation but requires template + script sections

React: Explicit state management with useState and setter functions

jQuery: Most verbose - manual DOM manipulation + separate HTML
